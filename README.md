# Claude Desktop Manager

A macOS menu bar app that brings **per-project isolation** to [Claude Desktop](https://claude.ai/download). Manage MCP servers, secrets, and individual tools for each of your projects — without touching JSON files.

## Why

Claude Desktop uses a single global MCP configuration. That means every project shares the same servers, the same tools, and the same API keys. Claude Desktop Manager fixes this by giving each project its own isolated MCP setup.

## Features

### Per-Project Isolation

Each project gets its own MCP configuration that activates when you select it. Switch between projects and Claude Desktop automatically picks up the right servers, tools, and credentials. No more global config collisions.

### Opinionated Project Structure

Projects follow a clean, predictable layout:

```
your-project/
├── .claude/
│   └── infra/
│       ├── .mcp.json            # your MCP server definitions
│       ├── .env                 # secrets (API keys, tokens)
│       └── gateway.config.json  # auto-generated, do not edit
└── .mcp.json                    # auto-generated, points Claude at the gateway
```

- **`.claude/infra/.mcp.json`** — you define your MCP servers here
- **`.claude/infra/.env`** — your secrets, loaded as environment variables per server
- **`.claude/infra/gateway.config.json`** — generated by the app, tracks enabled/disabled state and discovered tools
- **`.mcp.json` (project root)** — generated by the app, tells Claude Desktop to use the gateway

### Per-Project Secrets

Each project has its own `.claude/infra/.env` file. API keys and tokens stay scoped to the project that needs them. No shared credentials leaking across projects.

### Fine-Grained Tool Management

Don't just toggle servers on and off — toggle individual tools. Scan each server to discover its available tools, then enable only the ones you need. The gateway proxy only exposes what you've explicitly enabled.

### MCP Gateway Proxy

A lightweight Node.js process aggregates all your enabled servers into a single MCP endpoint. Claude Desktop connects to one gateway instead of N separate servers. The gateway:

- Connects to each enabled upstream MCP server
- Exposes only the tools you've enabled
- Proxies tool calls to the right upstream server
- Preserves original tool schemas (no type flattening)

### One-Click Apply

Toggle servers and tools in the menu bar, then click **Apply & Restart**. The app saves the config, restarts Claude Desktop, and you're running with the new setup.

## Prerequisites

- macOS 14 (Sonoma) or later
- [Node.js](https://nodejs.org) 18+
- [Swift](https://swift.org) 5.10+ (included with Xcode 15+)
- [Claude Desktop](https://claude.ai/download)

## Installation

```bash
git clone https://github.com/marks97/claude-desktop-manager.git
cd claude-desktop-manager
./install.sh
```

The install script will:
1. Install gateway npm dependencies
2. Build the Swift app in release mode
3. Create `ClaudeDesktopManager.app` in `/Applications`
4. Register a LaunchAgent so it starts at login
5. Launch the app

## Getting Started

### 1. Set up your project

Create the infra directory and MCP config in your project:

```bash
mkdir -p your-project/.claude/infra
```

Add your MCP servers to `.claude/infra/.mcp.json`:

```json
{
  "mcpServers": {
    "my-server": {
      "command": "npx",
      "args": ["-y", "@my/mcp-server"]
    }
  }
}
```

Add secrets to `.claude/infra/.env` (optional):

```
API_KEY=sk-...
DATABASE_URL=postgres://...
```

### 2. Add the project

Open Claude Desktop Manager from the menu bar and click **Add Project**. Select your project folder.

### 3. Scan and configure

Click **Scan Tools** to discover available tools from each server. Toggle servers and individual tools on or off as needed.

### 4. Apply

Click **Apply & Restart** to write the config and restart Claude Desktop with the new setup.
